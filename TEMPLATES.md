Resume Labs — Template Layout Guidelines

This document defines the resume template layout system for Resume Labs.

It does not cover app UI screens.
It covers the document layouts that should be implemented for resume templates.

The goal is to make templates:

* professional
* readable
* export-safe
* photo-safe
* ATS-aware where needed
* visually distinct without becoming gimmicky

⸻

1. Core Rules for All Templates

1.1 Page setup

* Page size: A4
* Safe margins: 32–40 pt on all sides
* Content must never sit too close to edges
* Long content must support multi-page overflow
* Section blocks must break cleanly across pages

1.2 Typography system

Each template may vary visually, but must still follow a controlled hierarchy.

Recommended type roles:

* Name / main title: 22–28 pt
* Section headings: 10.5–13 pt
* Role / degree titles: 11–13 pt
* Meta text: 9.5–10.5 pt
* Body text / bullets: 10–11 pt

1.3 Content rules

All templates should support these core sections:

* Personal Summary / Objective
* Work Experience
* Education
* Skills
* Projects
* Achievements / Awards
* Publications
* Interests
* Languages
* Reference
* Custom Sections

1.4 Photo rules

All templates must support both:

* with photo
* without photo

Photo rules:

* image should never overlap text
* text layout must reflow if photo exists
* divider lines must respect image bottom edge
* use circular or square photo only if the template style supports it
* fallback placeholder must be visually clean

1.5 Section behavior

Each template should support:

* hidden sections when empty
* renamed section labels
* reordered sections
* custom sections

1.6 Layout quality rules

Do not allow:

* overly decorative bars
* giant accent elements
* weak spacing rhythm
* cramped text blocks
* tiny preview-only layouts that fail in export
* random color usage without hierarchy

⸻

2. Template System Strategy

Resume Labs should not have random templates.
Each template should belong to a clear layout family.

Recommended families:

1. Classic
2. Modern
3. Executive
4. Minimal
5. Modern Clean
6. Modern Sidebar

Each template must differ in layout logic, not just color.

⸻

3. Classic Template Guidelines

3.1 Purpose

Use for:

* standard professional resumes
* broad job applications
* ATS-friendly preference
* clean corporate presentation

3.2 Layout structure

Top-down single-column layout

Order:

1. Header
2. Summary
3. Experience
4. Education
5. Skills
6. Additional sections

3.3 Header design

* left-aligned main name/title
* optional photo on top-right
* no decorative blocks
* thin divider line below header
* subtitle can be a small role line like “Professional Resume” or user job title

3.4 Section styling

* uppercase section titles
* subtle divider line beneath heading
* no cards
* no filled boxes
* content flows naturally like a real CV

3.5 Experience layout

For each job:

* Role title first
* Company and location second
* Date range third
* Bullet list below

Bullets:

* standard round bullet
* max visual noise: low
* line spacing readable

3.6 Education layout

For each item:

* Degree + field
* School
* Graduation date / GPA meta line

3.7 Skills layout

* simple inline list or wrapped plain chips with minimal borders
* avoid colorful pills in classic template

3.8 Visual tone

* conservative
* neutral
* professional
* print-friendly

3.9 Best use case

This should be the safest export template.

⸻

4. Modern Template Guidelines

4.1 Purpose

Use for:

* modern professional roles
* product, tech, digital jobs
* users who want clean but slightly more branded output

4.2 Layout structure

Single-column modern editorial layout

Order:

1. Accent header
2. Summary
3. Experience
4. Education
5. Skills
6. Additional sections

4.3 Header design

* horizontal accent bar at top
* strong left-aligned name/title
* optional photo on right
* subtitle under name
* divider placed below image/text bottom + spacing

4.4 Section styling

* section titles use accent color
* thin neutral divider line
* no bulky cards
* spacing slightly more open than classic

4.5 Experience layout

* role title in strong dark text
* company/location in muted text
* date range below
* bullets with accent-colored bullet marker if needed

4.6 Education layout

* clean academic block
* strong degree line
* muted school + date line

4.7 Skills layout

* inline grouped list preferred
* can use very light subtle pills if restrained
* avoid loud badges

4.8 Visual tone

* clean
* premium
* contemporary
* slightly branded

4.9 Best use case

Great default modern template for tech and design-adjacent applicants.

⸻

5. Executive Template Guidelines

5.1 Purpose

Use for:

* senior candidates
* leadership roles
* management / business resumes
* more polished authority-driven presentation

5.2 Layout structure

Structured single-column layout with stronger hierarchy

Order:

1. Premium header
2. Executive Summary
3. Professional Experience
4. Education
5. Core Competencies
6. Additional sections

5.3 Header design

* strong top accent rule or bar
* name/title large and confident
* optional photo right-aligned
* calm, premium spacing
* no flashy gimmicks

5.4 Section styling

* section titles can use dark blue / premium accent
* stronger heading tone than classic
* still restrained and document-like

5.5 Experience layout

* emphasize role impact
* slightly wider spacing between jobs
* bullets should feel strong and results-oriented
* suitable for leadership achievements

5.6 Skills layout

Rename as:

* Core Competencies
* Key Strengths
* Leadership Skills

Display as:

* inline list
* grouped columns
* refined tags

5.7 Visual tone

* premium
* senior
* structured
* calm authority

5.8 Best use case

For senior professionals, managers, consultants, executives.

⸻

6. Minimal Template Guidelines

6.1 Purpose

Use for:

* users who want ultra-clean, low-noise documents
* ATS-friendly but visually refined resumes
* academic / technical / understated roles

6.2 Layout structure

Pure minimal single-column layout

Order:

1. Minimal header
2. Summary
3. Experience
4. Education
5. Skills
6. Additional sections

6.3 Header design

* no top accent bar required
* just clean name/title + optional subtitle
* optional photo right-aligned if present
* single thin divider below

6.4 Section styling

* very subtle section titles
* thin divider
* lots of whitespace
* no decorative cards or boxes

6.5 Experience layout

* understated hierarchy
* dash bullets or very light bullets
* compact but readable spacing

6.6 Education layout

* minimal academic lines
* avoid visual heaviness

6.7 Skills layout

* plain inline list preferred
* no heavy styling

6.8 Visual tone

* understated
* elegant
* calm
* editorial minimalism

6.9 Best use case

For people who want a timeless, distraction-free resume.

⸻

7. Modern Clean Template Guidelines

7.1 Purpose

Use for:

* clean contemporary resumes
* users who want a modern document without strong branding
* polished mainstream professional resumes

7.2 Layout structure

Balanced modern single-column layout

Order:

1. Clean header
2. Professional Summary
3. Experience
4. Education
5. Skills
6. Additional sections

7.3 Header design

* strong name/title
* optional photo right-aligned
* no oversized decorative block
* neutral divider below

7.4 Section styling

* uppercase section labels
* subtle neutral dividers
* spacing-driven structure
* very clean transitions between sections

7.5 Experience layout

* modern but document-first
* role title prominent
* company and location muted
* bullets clean and evenly spaced

7.6 Skills layout

* inline list
* grouped list
* subtle wrap allowed

7.7 Visual tone

* clean
* controlled
* versatile
* highly usable

7.8 Best use case

This can be one of the best default templates for most users.

⸻

8. Modern Sidebar Template Guidelines

8.1 Purpose

Use for:

* visually distinctive resumes
* creative-tech users
* people who want a more designed look while keeping structure

8.2 Layout structure

Two-column layout with sidebar

Sidebar holds:

* photo
* short title / name block
* skills
* contact info
* small metadata sections

Main content holds:

* summary
* work experience
* education
* projects
* achievements

8.3 Sidebar design

* dark or accent sidebar
* clear contrast
* photo near top
* stacked compact section items
* keep sidebar text smaller but readable

8.4 Main content design

* bright neutral background
* strong content rhythm
* section headings in accent color
* readable line lengths

8.5 Critical layout rules

* sidebar content must never run into footer/overflow blindly
* main content must align well against sidebar start
* photo placement must not break the sidebar title area
* sidebar width must stay controlled

8.6 Skills layout in sidebar

* stacked list
* optionally grouped by category
* no giant pills

8.7 Visual tone

* modern
* structured
* visually stronger than single-column templates

8.8 Best use case

Good for users who want a more graphic resume without going full creative portfolio.

⸻

9. Cover Letter Layout Guidelines

Resume Labs should eventually support cover letter layouts that visually pair with resume templates.

9.1 Cover letter structure

1. Header
2. Sender details
3. Recipient block
4. Opening paragraph
5. Body paragraphs
6. Closing paragraph
7. Signature area

9.2 Pairing rule

Each resume family should have a matching cover letter family:

* Classic Resume → Classic Cover Letter
* Modern Resume → Modern Cover Letter
* Executive Resume → Executive Cover Letter
* Minimal Resume → Minimal Cover Letter
* Sidebar template may use a simplified matching cover letter, not full sidebar

9.3 Photo behavior

* cover letters should usually support optional small header photo only
* do not force sidebar/photo-heavy cover letters

⸻

10. Template Preview Guidelines

Preview cards in app should accurately reflect layout families.

10.1 Preview thumbnail requirements

Each thumbnail should clearly show:

* header style
* section spacing
* photo placement logic
* whether it is single-column or sidebar
* visual density

10.2 Template card info

Each template should have:

* template name
* layout category
* short one-line description

Example:

* Classic — Safe professional ATS layout
* Modern — Clean modern accent layout
* Executive — Premium leadership-oriented layout
* Minimal — Simple distraction-free format
* Modern Sidebar — Structured sidebar layout

⸻

11. Formatting Controls Per Template

Templates should support controlled formatting, but not unlimited chaos.

11.1 Global adjustable controls

Recommended:

* accent color
* body text size
* heading size
* name size
* line spacing
* font family
* section spacing
* photo on/off

11.2 Template-specific restrictions

Some template families should restrict certain settings.

Classic

* no heavy color customization
* only subtle accent options

Minimal

* keep styling constrained
* avoid loud colors

Executive

* allow premium dark blue / muted palette options

Sidebar

* allow sidebar color themes
* keep contrast readable

⸻

12. ATS and Print Safety

12.1 ATS-safe templates

Best ATS-safe templates:

* Classic
* Minimal
* Modern Clean

12.2 Moderate ATS-safe templates

* Modern
* Executive

12.3 Less ATS-safe but still usable

* Modern Sidebar

12.4 ATS safety rules

To improve ATS compatibility:

* keep text selectable
* avoid image-based text
* do not rely on tables for core parsing
* avoid decorative icons inside every text line
* keep section headings explicit
* maintain logical reading order

⸻

13. Shared Component System for Template Engine

All templates should be built from shared structural primitives.

13.1 Shared primitives

Implement reusable helpers for:

* photo rendering
* header rendering
* section heading rendering
* divider rendering
* work experience block
* education block
* body text block
* skill list block
* page break handling

13.2 Why this matters

Right now template files can become duplicated messes.
Shared helpers reduce:

* layout inconsistency
* photo bugs
* spacing bugs
* future maintenance cost

15. Final Template Quality Checklist

A template is only acceptable if:

* photo layout is safe
* no divider collides with image
* header is balanced
* typography hierarchy is clear
* section spacing is consistent
* long content does not break badly
* no decorative element feels oversized
* output looks like a real professional resume
* preview matches export closely

⸻

16. Final Recommendation

For Resume Labs, the template strategy should be:

* Classic for safety
* Modern Clean for default modern use
* Executive for senior candidates
* Minimal for understated users
* Sidebar as an advanced option, not the default

The biggest mistake would be building many templates that are only color variations.
Each template must be a real layout family with a clear purpose.
